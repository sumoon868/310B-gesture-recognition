# 昇腾310B 开发备忘

## 一、硬件连接

```
昇腾310B USB1 ──→ USB摄像头 (640×480)
昇腾310B USB2 ──→ CH340 USB-TTL
                     ├── TXD ──→ STM32 PA10 (USART1_RX)
                     ├── RXD ──→ STM32 PA9  (USART1_TX)
                     └── GND ──→ STM32 GND

STM32 ← ST-Link V2 (SWDIO/SWCLK/GND/3.3V 供电)
STM32 PA7 → [330Ω] → WS2812B DI
WS2812B 5V/GND → 外部5V/2A电源 + 1000μF电容
```

> ⚠️ CH340 TXD→PA10, RXD→PA9（交叉！）

---

## 二、STM32 通信协议

| 参数 | 值 |
|------|-----|
| 波特率 | 115200 |
| 数据位 | 8 |
| 校验 | 无 |
| 停止位 | 1 |
| 帧结尾 | `\r\n` 或 `\r` 或 `\n`（已兼容） |
| 响应 | 成功 `OK`，失败 `ERR:unknown command` |

### 命令表（昇腾侧发送）

| 命令 | 灯效 | 触发手势 |
|------|------|----------|
| `RAINBOW` | 彩虹流水 | like |
| `OFF` | 全灭 | palm, stop |
| `SOLID:255,0,0` | 红色常亮 | fist |
| `SOLID:255,255,255` | 白色常亮 | one |
| `BREATH:0,0,255` | 蓝色呼吸 | peace |
| `BREATH:255,0,255` | 紫色呼吸 | peace_inv |
| `CHASE:0,255,0` | 绿色追逐 | ok |
| `FLASH:255,0,0` | 红色闪烁 | dislike |
| `FLASH:255,255,0` | 黄色闪烁 | rock |
| `PULSE:128,0,255` | 紫色脉冲 | call |
| `SOLID:128,128,128` | 暗白常亮 | mute |
| `BRIGHT:50` | 亮度50% | (保留) |

## 四、Python 模块架构

```
gesture_led_main.py        ← 主入口，摄像头+推理+串口
    ├── serial_comm.py     ← SerialComm 类（串口收发）
    └── gesture_to_led.py  ← GestureMapper 类（手势→命令映射）
                               HAGRID_LABELS (34类)
                               GESTURE_TO_COMMAND 字典
```

### 4.1 serial_comm.py 核心接口
```python
comm = SerialComm(port='/dev/ttyUSB0')   # 不传port自动检测
comm.connect()                            # 打开串口，返回 True/False
comm.send_command('RAINBOW')              # 发送命令，返回响应字符串
comm.disconnect()                         # 关闭
```

### 4.2 GestureMapper 核心接口
```python
mapper = GestureMapper(debounce_ms=500)   # 500ms去抖动
cmd = mapper.map('like', 0.85)           # 返回 'RAINBOW' 或 None
```

### 4.3 GestureRecognizer 工作模式

| 模式 | 条件 | 行为 |
|------|------|------|
| ONNX推理 | 传入 `--model xxx.onnx` | CPU 真实推理 |
| 模拟模式 | 不传 `--model` | 随机手势（30%概率），测试通信链路 |

## 五、运行命令

```bash
# 最简运行（无模型，随机手势测通信）
python3 gesture_led_main.py

# 指定串口
python3 gesture_led_main.py --port /dev/ttyUSB0

# 带ONNX模型（真实手势识别）
python3 gesture_led_main.py --port /dev/ttyUSB0 --model yolov10n_gestures.onnx

# 指定摄像头
python3 gesture_led_main.py --port /dev/ttyUSB0 --camera 1
```

## 六、调试技巧

### 6.1 串口直发测试（不跑 Python 程序）
```bash
# 在昇腾310B终端直接发命令验证 STM32
echo -ne "RAINBOW\r\n" > /dev/ttyUSB0        # 灯带应出彩虹
echo -ne "SOLID:255,0,0\r\n" > /dev/ttyUSB0  # 灯带应全红
echo -ne "OFF\r\n" > /dev/ttyUSB0             # 灯带应熄灭

# 查看 STM32 返回
cat /dev/ttyUSB0 &                            # 后台监听
echo -ne "RAINBOW\r\n" > /dev/ttyUSB0        # 应收到 OK
kill %1                                       # 停止监听
```

### 6.2 串口列表
```bash
python3 -m serial.tools.list_ports
# 输出示例: /dev/ttyUSB0 - USB-SERIAL CH340 (COM10)
```

### 6.3 摄像头测试
```bash
python3 -c "import cv2; cap=cv2.VideoCapture(0); print(cap.read()[0])"
# 输出 True 表示摄像头正常
```

### 6.4 程序内手动测试
运行 `gesture_led_main.py` 后，按 **`t`** 键可在终端输入任意命令手动测试 STM32，无需做手势。

## 七、常见问题

| 问题 | 排查 |
|------|------|
| `SerialException: could not open port` | CH340 没插好 / 被其他程序占用 / 端口号不对 |
| 串口连上了但命令无效 | 检查 TXD→PA10, RXD→PA9 是否交叉（最常见的接错） |
| 摄像头打不开 | `ls /dev/video*` 确认设备存在，或换 `--camera 1` |
| `import acl` 失败 | CANN 环境没配置，`source set_env.sh` |
| 手势识别准确率低 | 调 `CONF_THRESHOLD`（默认0.5），或检查光照条件 |
| STM32 无响应 | STM32 是否上电、PC13 是否亮、串口线是否共地 |
| 第一颗灯珠颜色异常 | DI 端加 1kΩ 上拉电阻到 5V |

## 八、开发流程建议

```
第一步：昇腾310B 终端 echo 直发命令 → 灯带响应 ✓
第二步：python gesture_led_main.py（模拟模式）→ 随机手势触发灯效 ✓
第三步：加载 ONNX 模型 → 真实手势触发灯效 ✓
第四步：调优阈值/去抖动/预处理 → 精度达标 ✓
第五步：可选用 aiortc 加 WebRTC 推流
```
